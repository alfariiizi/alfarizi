## Overview

KurirGo is a shipping aggregator platform for sellers and general users who need to send packages through multiple expedition providers from one application.

## Business Context

The product is positioned around the same practical problem solved by shipping aggregators: users want access to shipping options, pricing, order creation, and tracking without managing each provider separately.

## My Role

I served as a freelance technical lead for the project, owning the engineering direction and delivery coordination across the system.

## Key Capabilities

- Expedition provider integration
- Shipping rate calculation
- Order lifecycle management
- Package tracking workflow
- Admin and operator workflows
- Release and delivery ownership
- Team code review and engineering standards

## Technical Direction

The platform uses Go for backend services and Next.js for the web application. The engineering direction focused on service boundaries, provider integration contracts, pricing flow, order state, and maintainable delivery practices for the team.

## Outcome / Impact

KurirGo represents end-to-end product engineering work across architecture, provider integration, operational workflows, and team delivery.
